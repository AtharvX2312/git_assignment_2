
Repositories: Containers for projects that store code, documentation, and version history.
​Branching: Allows developers to create isolated workspaces to develop features or fix bugs without affecting the main codebase.
​Pull Requests: A collaborative tool that lets contributors propose changes, receive feedback through code reviews, and discuss improvements before merging code.
​Issues: A tracking system used to report bugs, request new features, and manage project tasks.
​GitHub Actions: An automation tool that enables continuous integration and continuous delivery (CI/CD) directly within your repository.
​Discussions: A dedicated forum for community conversations, Q&A, and announcements, separate from task-oriented issues.
​Projects: A project management tool that uses Kanban-style boards to organize, track, and prioritize work.
​Insights: A tab that provides data on repository activity, including commit history, contributor stats, and dependency graphs.
